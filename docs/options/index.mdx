## Configuration Methods

### CLI Options

Configure Packen directly from the command line:

```bash
# Basic build with options
packem build --minify --sourcemap

# Watch mode with custom output
packem build --watch --outDir dist

# Production build with all optimizations
packem build --minify --clean --target es2020
```

### Configuration File

Use `packem.config.ts` for complex configurations:

```typescript
import { defineConfig } from '@visulima/packem/config'

export default defineConfig({
  // Entry points
  entry: 'src/index.ts',
  
  // Transformer selection
  transformer: 'esbuild',
  
  // Output configuration
  rollup: {
    output: {
      format: 'esm',
      dir: 'dist'
    }
  },
  
  // CSS processing
  css: {
    extract: true,
    modules: true
  }
})
```

### Programmatic API

Configure Packen in your Node.js scripts:

```typescript
import { packem } from '@visulima/packem'

await packem('./src', {
  mode: 'build',
  environment: 'production',
  minify: true,
  sourcemap: true
})
```

## Default Values

Packen provides sensible defaults for most options:

- **Entry**: Automatically detected from `package.json` exports
- **Output Format**: ESM for modern packages, CJS for compatibility
- **Transformer**: esbuild for optimal performance
- **Target**: Determined from `package.json` engines field
- **Source Maps**: Enabled in development, disabled in production
- **Minification**: Disabled by default, enabled for production builds

## Environment-Specific Configuration

Configure different options for different environments:

```typescript
export default defineConfig({
  transformer: process.env.NODE_ENV === 'development' ? 'esbuild' : 'swc',
  
  rollup: {
    sourcemap: process.env.NODE_ENV === 'development',
    minify: process.env.NODE_ENV === 'production'
  }
})
```

## Validation and Type Safety

All configuration options are fully typed and validated:

```typescript
// TypeScript will provide autocomplete and type checking
export default defineConfig({
  transformer: 'esbuild', // ✅ Valid
  transformer: 'invalid', // ❌ TypeScript error
  
  rollup: {
    format: 'esm' // ✅ Valid
    format: 'invalid' // ❌ TypeScript error
  }
})
```
