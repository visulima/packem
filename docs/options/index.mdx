# Configuration Options

All options are set in `packem.config.ts` using `defineConfig`. This page lists every available option grouped by category. Click through to the individual pages for examples and details.

```typescript
import { defineConfig } from '@visulima/packem/config'
import transformer from '@visulima/packem/transformer/esbuild'

export default defineConfig({
  transformer,
  // ...options below
})
```

## Input and output

| Option | Description |
|--------|-------------|
| [entry](/docs/options/entry) | Configure entry points and automatic detection |
| [config-file](/docs/options/config-file) | Config file setup, formats, and environment variables |
| [output-directory](/docs/options/output-directory) | Control where built files are written |
| [output-format](/docs/options/output-format) | ESM, CJS, IIFE output formats |
| [output-extension-map](/docs/options/output-extension-map) | Custom file extensions for output files |
| [source-dir](/docs/options/source-dir) | Source directory for entry point resolution |
| [root-dir](/docs/options/root-dir) | Project root directory |
| [name](/docs/options/name) | Package name override |

|--------|-------------|

| [minify](/docs/options/minify) | Minification with terser, esbuild, or oxc |
| [source-maps](/docs/options/source-maps) | Source map generation |
| [externals](/docs/options/externals) | External dependency handling |
| [alias](/docs/options/alias) | Module path aliases |
| [barrel](/docs/options/barrel) | Barrel file optimization |
